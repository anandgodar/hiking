## ðŸ”§ Data Gathering Workflow

### Step 1: Research (15 minutes per trail)

**Official Sources (Always use these first):**
1. **National Park Service** - For Yosemite, Sequoia, Joshua Tree, Death Valley
   - URL: nps.gov/[park]/planyourvisit/hiking.htm
   - Provides: Distance, elevation gain, difficulty, trailhead location
   - **Example:** https://www.nps.gov/yose/planyourvisit/halfdome.htm

2. **California State Parks** - For state park trails
   - URL: parks.ca.gov
   - Provides: Trail maps, parking info, fees

3. **USDA Forest Service** - For National Forest trails
   - URL: fs.usda.gov/main/angeles (or Cleveland, Eldorado, Shasta-Trinity, etc.)
   - Provides: Trail details, wilderness permits, conditions

4. **Bay Area Parks** - For SF Bay Area trails
   - URL: ebparks.org, openspace.org, parks.smcgov.org
   - Provides: Trail maps, parking, hours

**GPS & Elevation:**
- **Google Maps** - Get lat/lon coordinates (right-click â†’ "What's here?")
- **CalTopo** (caltopo.com) - Topographic elevation data, GPX tracks
- **USGS TopoView** - Official USGS topo maps

**Photos:**
- **Unsplash.com** - Search "[trail name] california" or "[park name]"
- Free high-quality photos, already optimized
- Use query: `?auto=format&fit=crop&q=80&w=2000`

---

### Step 2: Create JSON File (20-30 minutes per trail)

**Naming Convention:**
```
[trail-name]-[region].json

Examples:
- half-dome-yosemite.json
- potato-chip-rock.json
- mission-peak.json
- mount-wilson-la.json
```

**Minimum Required Fields:**
- name, slug, state, state_slug, elevation, lat, lon
- mountain_hero (photo URL)
- tags (array)
- trails array with at least one trail object
  - name, difficulty, type
  - stats: distance, gain, time, difficulty
  - parking_info

**Nice-to-Have (add if time permits):**
- geo.markers (start/summit coordinates)
- geo.path (trail route coordinates)
- geo.chart (elevation profile)
- features array
- generated_description
- page_content (FAQs, seasonal guide, safety)
- seo (meta title/description)
